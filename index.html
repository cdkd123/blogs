u3d动画状态机

动画状态机的设置

顾名思义，动画状态机，就是每个状态对应不同动画。这里涉及到动画，状态机2个概念。动画组件怎么制作，我们之前讲过。而状态机，就是一个对象有很多个状态，比如一个人，他的动作，就是个状态；他有跑动状态，走路状态，跳跃状态，倒下状态等。每个状态满足一定条件都可以变换到其他状态，这种状态切换系统，叫状态机。既有动画，又有状态机，就总称为动画状态机。

下面就讲一下怎么制作这样一个动画状态机。

  1. 给组件增加一个AnimationControllor对象.
  2. 创建动画状态。
    • window -> Animator
    • Create State -> Empty 创建状态。 多创建几个。

  3. 动画状态过渡线的创建

    • Empty 右键 -> Make Transition

    • 移动鼠标，即可连上其他状态。

每个State，都可以修改name, animation, 状态切换条件等。

首先，我们可以新增加几个变量，作为状态切换条件的控制变量。

在左边 Paramter 窗口，我们可以点 +号 新增 Float， Boolean， Int， Trigger 四种变量。比如，创建isRun Boolean 形的变量，然后再选择2个状态之间的切换线，在Inspector视图，Transition一列，新增 isRun ，即可以设置 isRun 控制状态切换。

最终，所有的这些状态，互相连接起来!

国外的计算机科学与技术

最近扫了一眼Coursera上的伊利诺伊州立大学的计算机专业，有些惊讶。他们清楚的定义自己是”数据科学领域的计算机科学硕士“专业。他们教授的已经是纯软件，纯数据方向的知识了。

在人才需求方面，网页设计，移动应用会比较多，游戏开发需求也不少，但，其实说，这些领域最近两年人才快速增长，如果现在入学就学，毕业未必不会人才过剩呢。

所以，要我说，去报“AI”方向吧。AI不会人才过剩？我认为，

  1. 学习成本高，回报也高。并不是两三个月就能学会的。
  2. 非常适合学生在校学习。
  3. 高端AI人才的缺口非常大。（统计数字：）
  4. AI是国家战略发展方向。所以有前途。

但是国内的AI技术，现在不论是媒体，还是科技公司对外宣传，感觉都吹的有点过了。

和普通函数，匿名函数的区别

上面提供的 lambda 表达式语法缺少的一个东西是指定函数的返回类型的能力。在大多数情况下，这是不必要的。因为返回类型可以自动推断出来。
然而，如果确实需要显式指定，可以使用另一种语法：匿名函数。

  1. 普通函数声明：

函数名称，参数名，参数类型，返回值类型，函数体统统写清楚了。表达清晰，代码规范。这就是普通函数的声明。

  1. 匿名函数的声明：

匿名函数不需要声明函数名，参数名，甚至返回值类型。 直接写函数体就可以了：

请注意，匿名函数参数总是在括号内传递。允许将函数留在圆括号外的简写语法仅适用于 lambda 表达式。

总结一下它们的区别：

  1. 普通函数，需要先声明“函数名，参数，返回类型”，再使用。
  2. 匿名函数，省掉了函数名称，但依然要指定”参数类型，名字，返回类型。
  3. lambda表达式，根本不需要声明，调用时，符合调用者声明的函数类型即可。
