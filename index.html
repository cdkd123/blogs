遗憾的婚礼见证人

前记

华仔是我以前的室友，和小伙相处两年挺久，长得帅气，又有人缘，挺受姑娘欢迎一伙计。
大约半年前回到我省省会。工作半年，突然宣布结婚。惊喜之余，真的觉得在这方面真的差他2个德华啊。大年初六他老家办婚礼，去他家路还挺好走的。

一件小事

当天到小镇十字路口，开着新买的车接到他家，正好赶上一场酒席 ^_^ ，蹭了一顿！
李还在路上，说好的一起出席他的婚礼的。这伙计天生自来熟，不怕生人，非常有头脑，口才也好，还是本地人。

华仔他爸，人挺好，看我俩面善，长得帅气，又都是带眼镜的，一看就是知识分子，决定交给我俩一个重要任务 – 第二天的婚礼司仪！我连忙吱吱唔唔推脱:”这个，哎，没经验，不好吧？。。之类的。”, 他爸就说：“没关系的，就简单的按照流程来，和大家寒暄几下就好啦。 ”

这样赶鸭子上架，就定我俩上台了。

当天晚上，女方亲戚都已经来到宾馆，第二天准备好好庆祝的。人，非常之多。心有点慌。

晚上睡觉时，我和李商量：“小李，你是本地人，对这边礼仪表熟悉，要不明天你来主持？”还打着王者荣耀的他，嗯了一声。

一件小事续

过去的三年，每年都参加了一场婚礼，好像都是他们荆州这一带！

传统习俗是，早上5点起早床去迎接新娘吧。

睡到早上8点！！！

于是，带着新娘绕全镇一圈，敲锣打鼓，放鞭炮，到了进门的那一刻了。

话筒到了李手中，李道：“各位叔叔阿姨，伯伯奶奶，哥哥姐姐弟弟妹妹们，大家新年好！
首先，在这里要欢迎各位亲朋好友的到来，参加xx和xx的婚礼。”，他带点方言，众亲朋开始鼓掌，然后他喵了喵大堂上写的流程牌子，“第一项，新郎发言！”，华仔各种感谢词之类讲完，“第二项，新娘发言”，新娘讲类似的话。

“第三项，有请证婚人发言”。有个大叔推举一个大伯出来，那大伯，赶紧摆手道：“不，不行不行！”。刚好他爸看我站在旁边，说：“小伙子，你来吧！”。

此时，我内心OS：我还是个孩子啊！爸！

在蒙圈了三秒后，话筒到了我的手里：“那个，今天。。。我们见证了华仔和xxx的这次婚礼哈，那么非常棒～下一项！”

众人有些困惑！我惊觉冷场了，要搞砸了！！！

正好李，结果话筒道：“那个，这伙计太紧张，嘴有些不利索哈，他的意思就是，他见证了这对新人从相识，相知，相爱，到走进婚姻殿堂，啊。。。（可能想起来，才回来半年，怎么可能见证？）就是一起玩过游戏，在王者荣耀里一起杀过敌，做过好队友啦！所以，恭喜他们！”。

在旁边的我表示震惊了。这也行～

于是，婚礼顺利的进行下去。

u3d动画状态机

动画状态机的设置

顾名思义，动画状态机，就是每个状态对应不同动画。这里涉及到动画，状态机2个概念。动画组件怎么制作，我们之前讲过。而状态机，就是一个对象有很多个状态，比如一个人，他的动作，就是个状态；他有跑动状态，走路状态，跳跃状态，倒下状态等。每个状态满足一定条件都可以变换到其他状态，这种状态切换系统，叫状态机。既有动画，又有状态机，就总称为动画状态机。

下面就讲一下怎么制作这样一个动画状态机。

  1. 给组件增加一个AnimationControllor对象.
  2. 创建动画状态。
    • window -> Animator
    • Create State -> Empty 创建状态。 多创建几个。

  3. 动画状态过渡线的创建

    • Empty 右键 -> Make Transition

    • 移动鼠标，即可连上其他状态。

每个State，都可以修改name, animation, 状态切换条件等。

首先，我们可以新增加几个变量，作为状态切换条件的控制变量。

在左边 Paramter 窗口，我们可以点 +号 新增 Float， Boolean， Int， Trigger 四种变量。比如，创建isRun Boolean 形的变量，然后再选择2个状态之间的切换线，在Inspector视图，Transition一列，新增 isRun ，即可以设置 isRun 控制状态切换。

最终，所有的这些状态，互相连接起来!

和普通函数，匿名函数的区别

上面提供的 lambda 表达式语法缺少的一个东西是指定函数的返回类型的能力。在大多数情况下，这是不必要的。因为返回类型可以自动推断出来。
然而，如果确实需要显式指定，可以使用另一种语法：匿名函数。

  1. 普通函数声明：

函数名称，参数名，参数类型，返回值类型，函数体统统写清楚了。表达清晰，代码规范。这就是普通函数的声明。

  1. 匿名函数的声明：

匿名函数不需要声明函数名，参数名，甚至返回值类型。 直接写函数体就可以了：

请注意，匿名函数参数总是在括号内传递。允许将函数留在圆括号外的简写语法仅适用于 lambda 表达式。

总结一下它们的区别：

  1. 普通函数，需要先声明“函数名，参数，返回类型”，再使用。
  2. 匿名函数，省掉了函数名称，但依然要指定”参数类型，名字，返回类型。
  3. lambda表达式，根本不需要声明，调用时，符合调用者声明的函数类型即可。

这是一个在线练习网站，可以针对所有kotlin语法知识点进行刻意练习，这个项目叫”kotlin-koans“，就如我们小学语文练习完形填空，一个小任务，会描述”任务目标“，在哪里修改代码，甚至测试代码都帮你写好了。如果觉得写不出来答案，或者写得不对，可随时切换到完成版本的分支，查看答案。
