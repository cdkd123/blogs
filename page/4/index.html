gitpages

事先说明

  • 之前用wordpress搭配新网域名，蛮好的，就是wordpress自身不支持markdown语法。
    不论装插件啦，从markdown编辑器生成静态网页啦，效果都不是很好。同时介绍gitpages还不错，就鼓捣了一下。然而，鼓捣了一个下午的时间，就卡在，如何不添加www也能访问gitpages个人主页这个问题上了。

这里两部分gitpages使用篇，和新网dns解析配置篇，介绍如何搭建这样一个小型个人博客。

gitpages使用套路

gitpages其实蛮好用。你上github网站上，自己新建个仓库。一步一步走下去。这样一个个人网站其实就搭成了。

如下:
登录进入github网站。
点击右上角的创建新仓库:

然后就进入创建仓库界面.仓库名字起个特别点的。例如username/liru， 接着，就是进入Settings界面，我们会看到Settings，Features，Merge button，GitHub Pages等项目。
让我们把目光停留在GitHub Pages，首先，选Source，只能选master branch对吧？ok,确定。然后，点save。 gitHub Pages 是不是出现了：Your site is ready to be published at https://username.github.io/liru/这样的字样？

接着，在浏览器里，输入https://username.github.io/liru/,会车，看看，不是看到自己的readMe.md 文件？哦，忘了说了，前提是你创建仓库的时候勾选了创建这个ReadMe.md 文件，没关系，现在创建也来得及。

新网dns解析配置套路

接下来咱就到第二步了.为什么是新网，不是万网，godaddy？因为一年前就买了这个域名啦！当然是原谅自己sb啦！（补充，可以转移出去）

然后这一下午时间就在搞这一件事了。http://域名.com为什么访问不了?

大致原因有:

  1. 你的url包含了破折号—。记得在gitpages的帮助指南里有提到。

    If the URL for your Pages site contains a username or organization name that begins or ends in a dash, or contains consecutive dashes, then people browsing with Linux will receive a server error when they visit the site. To fix this, change your GitHub username to remove non-alphanumeric characters. For instructions on how to do this, see “Changing your GitHub username.”

  2. 新网CNAME设置不对.正确的配置的应该是这样的:


然后在刚才的Settings里，GitHub Pages有个Custom domain,输入你的域名domain.com,就可以了。

新网配置的A记录，是10分钟后生效。所以10分钟后再访问下你的域名吧。

这是一个基础，后续咱们好要搞事情。先这样!

final，static，violate, synchronize关键词的意思？

final修饰类，表示类不可以继承。修饰变量，变量不可以被更改。修饰方法，方法不能被重写。
static 修饰类，表示类可以直接当做一个单独类使用，如果修饰是内部类，无需实例化外部类，可以直接new A.B()实例化。
不能修饰外部类。

内部类可以写静态变量吗？

安卓的类加载机制是怎样的？

  • DexClassLoader
    DexClassLoader用来加载jar、apk，其实还包括zip文件或者直接加载dex文件，它可以被用来执行未安装的代码或者未被应用加载过的代码。
  • PathClassLoader
    PathClassLoader被用来加载本地文件系统上的文件或目录，但不能从网络上加载，关键是它被用来加载系统类和我们的应用程序.
